Main File Structure

All the file are well organized, its so easy to work with the template.
First Unzip the mainFile.
Content: You will find 2 folder and 9 other files.
In the src folder there are 5 folders and 3 other files.
Component Directory: In the component folder there are the components of the template that are used all around and from here the components can be modifed according to your need.
Data Directory: In the data folder there is Data.tsx file in which there is all mock data used in the template.
Pages Directory: In the pages folder there are all the pages for the template. Here all the functional components are used as page element which is routed in the App.tsx using react router dom.
Redux Directory: Inside the redux folder in src which contains the redux state management store and functions.
Types Directory: In the types folder there is index.ts file in which there is all data types that can be used in the template via importing. It reduces multiple types declartion when prop drilling.
Public Directory: In the public folder there are static image fontawesome css, scss, images and webfonts.
Other Files: The rest of the files are .eslintrc.cjs, .gitignore, index.html, package-lock.json, package.json, readme, tsconfig.json, tsconfig.node.json and vite.config.ts file.


Environment Setup

In order to use or run Organiyo on your system, Firstly, you have to setup the React environment in the system, Follow the following steps to make your system compatible for using Organiyo or any other React project.

1. Install Node.js
Node.js is a JavaScript run time built needed to run a React App. You can install the recommended version of Node.js from the official website https://nodejs.org/en/

Steps to setup the project on the system:

Step 1 - Open The main-file: Open the main-file. You can either open the file in vs-code or any code editor of your choice and use the built-in terminal or you can also open the main-file in file manager and open a terminal or git bash in the root of the main-file.
Step 2 - Install Node Modules: To run Organiyo in local system you will need to install the node modules. You can type "npm install" command in terminal of the main-file's root.
Step 3 - Running In Local Server: Use "npm run dev" in the terminal to run the project in local server. Most common local server is http://localhost:5173/.
Step 4 - Creating A Production Build: With "npm run build" command you can build a product build of Organiyo. If any problem arises in deployment you can try to build in local and by doing so you can identify the problem as running build command gives out specific error message if any present. You can preview the production build by running command "npm run preview".

Deploying in Cpanel

1. Install Node Module: Run the command `npm install` in the project directory to install the dependencies of the Organiyo. This will install the required node modules for the project.
2. Build Organiyo: Run the command `npm run build` in the project directory to create a production-ready build of the Organiyo. This will generate optimized and bundled files in a `dist` directory.
3. Prepare your files: Select all the files in the dist directory. Zip the selected file and will be uploading it to the root directory of the domain created for Organiyo in Cpanel.
3. Access cPanel: Log in to your cPanel account provided by your hosting provider.
4. File Manager: Locate the "File Manager" or "Files" section in cPanel and open it.
5. Select your domain or subdomain: Choose the domain or subdomain where you want to deploy Organiyo. This will open the File Manager for that specific location.
6. Upload files: Inside the File Manager, click the "Upload" button or option and upload the previously zipped file. You can drag and drop the files or use the file upload interface.
7. Extract files (if necessary): If you uploaded a compressed file, such as a ZIP archive, select it in the File Manager and choose the "Extract" option to extract the files and folders.
8. Verify files: Ensure that all the files and folders from Organiyo are now present in the desired location on the server.
9. Access your Organiyo: After extracting the file in the file manager in the root directory of the domain, you should be able to access Organiyo by visiting your domain or subdomain in a web browser.

Steps to install and start in local server

1. Unzip the "mainFile" and you can open commad prompt in the mainFile and use command "npm install" to install node modules.
2. After installing node modules you can type the command "npm run dev" to open the web app in local server.
3. For production build use the command "npm run build".

Redux Toolkit:Streamlined State Management

Centralized State Handling:

The Redux Toolkit, housed within the src/redux directory, serves as the nerve center for managing the application's global state.
This centralized approach eliminates the need for excessive prop drilling and promotes a clear, single source of truth for state data.

Accessing State and Dispatching Actions:

Components seamlessly tap into the Redux store's state and dispatch actions using two custom hooks:
useAppDispatch: Dispatches actions to trigger state updates.
useAppSelector: Selects specific slices of state for component rendering.

Key Advantages:

Enhanced Performance: Redux Toolkit's streamlined approach often leads to reduced code complexity and improved performance.
Single Source of Truth: A single Redux store maintains the application's state, ensuring consistency and predictability.
Clear Data Flow: The unidirectional data flow pattern simplifies reasoning about state changes and debugging.
Well-Documented Reducer Slices: Comprehensive documentation within reducer slices fosters understanding and maintainability.

Data

Types

The types folder contains index.ts file that contains type declarations that are used in multiple components to reduce extra code writing.
Customization: You can add or update any type declaration in the index.ts file.

Pages Structure

In the pages folder there are multiple .tsx files which acts as pages for the template. In the App.tsx file in the root of src folder routing is created using react router dom. For a route we need to define a route and assign a main component for that route which are functional components in pages directory. For dynamic route we used slug for better user experience.
Customization: You can customize the pages of Organiyo in this file directory. You can add or remove components by mounting or unmounting in the .tsx file in the pages directory.
